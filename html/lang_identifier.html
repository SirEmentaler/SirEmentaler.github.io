Names and identifiers

An identifier is a sequence of letters, digits, and underscores of arbitrary length, beginning with a non-digit. Identifiers are case-sensitive, i.e. lowercase and uppercase letters are considered distinct.

Identifiers that aren't keywords can be used to name variables, functions, types, enumerators, namespaces, virtual properties, and other entities, during their declaration. An identifier that names a variable, a function, an enumerator, can be used as an expression. Such an expression returns the entity named by the identifier and has no side effects. An identifier that names a virtual property can be used to form a virtual property access expression whose value and side effects depend on context, see property accessors for details.

A name refers to an entity and is one of:

  • an identifier;
  • a class name preceded by the character ~ (a destructor name);
  • a template name followed by its argument list.
